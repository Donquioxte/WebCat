Web application Critical

Authentication bypass in a payment processing platform

A fintech startup commissioned a web application assessment ahead of a Series A fundraise. The engagement identified a chained vulnerability — an insecure direct object reference combined with a broken access control flaw — that allowed unauthenticated access to customer transaction records. The issue stemmed from an API endpoint that validated the session token but did not verify whether the authenticated user owned the requested resource.

A full remediation roadmap was delivered within 48 hours of discovery, and the retest confirmed all findings were resolved before the client's investor due diligence process began.

Network Critical

Internal network compromise via unpatched VPN appliance

A healthcare SaaS company engaged for an external network assessment ahead of a compliance audit. An unpatched VPN appliance exposed on the perimeter was identified as exploitable using a known CVE, providing an initial foothold on the internal network. From there, weak credential policies and a flat network architecture — no segmentation between clinical systems and general IT infrastructure — allowed lateral movement to a database server containing patient records.

The engagement report included a network segmentation recommendation alongside the individual findings, which the client's infrastructure team implemented over the following month.

Social engineering High

Phishing simulation reveals significant staff exposure across departments

An e-commerce platform with 80 staff requested a phishing simulation to assess security awareness levels ahead of a company-wide training rollout. Three phishing scenarios were designed — a generic IT helpdesk password reset, a targeted spear-phish impersonating the CEO, and a fake package delivery notification. Click rates and credential submission rates were tracked per department without capturing any real credentials.

Results revealed significant variation between departments — finance and operations teams showed the highest click rates, while the engineering team performed significantly better. Findings were used to tailor the client's awareness training programme by department rather than delivering a one-size-fits-all curriculum.

Web application High

Mass assignment and BOLA vulnerabilities in a B2B SaaS API

A B2B logistics platform engaged for a focused API security assessment. The API, built on a Node.js backend, was found to have a mass assignment vulnerability in the account update endpoint — allowing clients to promote their own accounts to administrator level by including undocumented fields in the request body. A separate BOLA (Broken Object Level Authorization) finding allowed one tenant to enumerate and read shipment records belonging to other tenants by manipulating resource IDs in the URL.

Both findings were remediated within 48 hours. The retest confirmed the mass assignment fix was complete, though the initial BOLA remediation was found to have missed one secondary endpoint, which was flagged and subsequently fixed.
