Toolkit

Tools I reach for on every engagement

Organised by category — recon, web application testing, network, and exploitation.

Amass
Recon
In-depth subdomain enumeration using passive and active techniques — DNS brute-forcing, certificate transparency logs, scraping, and API integrations. My go-to for building an initial subdomain list.
Linux / macOS / Windows GitHub ↗
httpx
Recon
Fast, multi-purpose HTTP toolkit for probing live hosts, detecting technology stacks, following redirects, and filtering large subdomain lists down to what's actually responding.
Linux / macOS / Windows GitHub ↗
ffuf
Recon
Fast web fuzzer for directory and file discovery, virtual host enumeration, parameter fuzzing, and more. Highly configurable and significantly faster than most alternatives.
Linux / macOS / Windows GitHub ↗
Burp Suite
Web app
The industry-standard intercepting proxy for web application testing. Used for every web engagement — traffic interception, active scanning, manual testing, and Intruder/Repeater-based vulnerability probing.
Linux / macOS / Windows Website ↗
Nuclei
Web app
Template-based vulnerability scanner from ProjectDiscovery. Excellent for rapidly identifying known CVEs, exposed panels, misconfigurations, and default credentials across large host lists.
Linux / macOS / Windows GitHub ↗
Arjun
Web app
HTTP parameter discovery tool that finds hidden, unlisted parameters accepted by web endpoints. Invaluable for surfacing forgotten parameters that are often missing proper input validation.
Linux / macOS GitHub ↗
sqlmap
Web app
Automated SQL injection detection and exploitation tool. Handles error-based, blind, time-based, and out-of-band injection, with built-in support for database enumeration and credential extraction.
Linux / macOS / Windows GitHub ↗
Nmap
Network
The standard for port scanning and network discovery. Used at the start of every network engagement to map open services, identify OS fingerprints, and detect service versions for further investigation.
Linux / macOS / Windows Website ↗
Netexec
Network
Swiss-army knife for Windows network assessment — SMB enumeration, credential validation, pass-the-hash attacks, remote command execution, and much more. The modern successor to CrackMapExec.
Linux GitHub ↗
Metasploit
Exploit
The most widely used penetration testing framework. Used primarily for exploit execution, payload generation, and post-exploitation — particularly useful for internal network engagements where known CVEs apply.
Linux / macOS / Windows Website ↗
BloodHound
Exploit
Active Directory attack path visualisation tool. Collects domain data via SharpHound and maps privilege escalation paths — nested groups, delegation misconfigurations, and ACL abuse — as a graph. Essential for any AD engagement.
Linux / macOS / Windows GitHub ↗
theHarvester
OSINT
Passive OSINT tool for gathering email addresses, subdomains, employee names, and open ports from public sources — search engines, Shodan, LinkedIn, and more. Good for pre-engagement recon on an organisation.
Linux / macOS GitHub ↗
Shodan
OSINT
Search engine for internet-connected devices. Useful for finding exposed infrastructure, identifying service versions, and discovering assets the client may not know are publicly reachable before active testing begins.
Web / CLI Website ↗
Ghostwriter
Reporting
Open-source engagement management and reporting platform built specifically for pen testers. Handles finding tracking, report generation, and client management in one place, keeping notes organised across long engagements.
Linux (self-hosted) GitHub ↗
Scripts

Things I've built

Small scripts I've written to automate repetitive tasks during testing. Nothing production-grade — just useful utilities.

subdomain-live-check.sh
Pipes a subdomain list through httpx with sensible defaults — status codes, titles, tech detection, and response size — and outputs a clean, filtered file ready for the next recon stage. Skips the manual flag-typing step every single time.
Bash · ~40 lines
View on GitHub
jwt-decode-check.py
Takes a JWT from stdin, decodes and pretty-prints the header and payload, checks for common misconfigurations (none algorithm, missing exp claim, weak algorithm), and outputs a quick summary. Useful as a first-pass check before reaching for jwt_tool.
Python · ~80 lines
View on GitHub
idor-sweep.py
Given a URL template with a numeric ID and two Burp session cookies, sweeps a range of IDs with both sessions and flags any responses where the lower-privilege session returns the same body length as the higher-privilege one. Quick IDOR triage before setting up Authorize.
Python · ~120 lines

Learning

Resources I keep coming back to

Platforms, references, and communities that have been genuinely useful while working through offensive security as a student.

Practice labs
HackTheBox
Realistic machines and Pro Labs covering web, network, and Active Directory attack paths. Most of the CTF write-ups on this blog come from here.
hackthebox.com ↗
Practice labs
PortSwigger Web Security Academy
Free, structured learning for web application vulnerabilities with interactive labs. The best free resource for methodical web security skill-building — the IDOR, SSRF, and JWT labs here are excellent.
portswigger.net ↗
Reference
HackTricks
Comprehensive, community-maintained reference for attack techniques across web, network, Active Directory, and more. My first stop when I need to refresh on a specific exploitation technique mid-engagement.
book.hacktricks.xyz ↗
Reference
OWASP Testing Guide
The definitive reference for web application security testing methodology. Not light reading, but an invaluable structured checklist for ensuring coverage across an engagement.
owasp.org ↗
Wordlists
SecLists
The most comprehensive collection of security-focused wordlists available — directory names, passwords, usernames, payloads, and more. Installed on every machine I test from.
GitHub ↗
Community
0xdf hacks stuff
Detailed, well-written HackTheBox write-ups from one of the community's best. Useful both for machines I've already done (comparing approaches) and as a reference for techniques I haven't encountered yet.
0xdf.gitlab.io ↗
